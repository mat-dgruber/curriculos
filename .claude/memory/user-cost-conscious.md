User is cost-conscious when it comes to infrastructure. Rejected Railway free tier (0.5GB storage, $1/month cap) as too limiting for production. Chose Oracle Cloud Always Free (VM ARM with 200GB storage) over paid alternatives like Google Cloud ($6-7/month) or Railway Hobby ($5/month).

**Key traits:**
- Willing to invest time in self-hosting (Docker, VM setup) to save money
- Researches alternatives thoroughly — asked about Supabase, Neon, Firebase, Google Cloud, Oracle Cloud before deciding
- Prefers "free forever" (Oracle Always Free) over trials that expire (Google Cloud 1-month free)
- Understands the tradeoff: self-hosting means more ops work, but values $0 cost

**How to apply:** When recommending infrastructure or tools, always lead with free-tier options. Mention paid alternatives only as "when you outgrow free tier" paths. Don't suggest services with small free tiers that could be easily exhausted.
