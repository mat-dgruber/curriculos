# Incidente 2026-06-22 — OOM leva jobhunter a 502/CORS

## TL;DR
Dashboard caiu com 502 mostrado ao browser como erro de CORS. Causa real: container `jobhunter` ficou ~8h travado (`FailingStreak: 28787`) e nginx passou a devolver 502. O "erro de CORS" era sintoma (nginx nao envia `Access-Control-Allow-Origin` em 502). `docker restart jobhunter` resolveu.

## Causa raiz
* VM Oracle Always Free estava usando apenas **956 MiB RAM** (doc dizia 8 GB — desatualizado).
* **Sem swap** no host → qualquer pico do Chromium/Playwright durante varredura leva a OOM-kill direto.
* `restart: unless-stopped` do docker-compose v1 não foi suficiente porque o container nao saiu — uvicorn travou / OOM manter o processo em estado quebrado (containers sao killados depois, mas com delay).

## Sintomas no browser
```
Failed to load resource: net::ERR_FAILED
Access to fetch ... has been blocked by CORS policy
502 Bad Gateway
Server: nginx/1.18.0 (Ubuntu)
Origin: https://hotel-cittari.web.app
```

## Mitigaçoes aplicadas (mesmo dia)
1. VM: `/swapfile` de 2 GB + sysctl `swappiness=10`, `overcommit_memory=1`.
2. `docker-compose.yml`: `mem_reservation: 500m`, `mem_limit: 700m`, `pids_limit: 200` → OOM-kill limpo, reinicio automatico via `restart: unless-stopped`.
3. `.env`: `PLAYWRIGHT_SLOW_MO=0` (estava 100; era pausa artificial desnecessaria em prod).
4. Logging driver com cap 3×10 MB (`max-size`/`max-file`).
